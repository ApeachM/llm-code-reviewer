## 1. 프롬프팅 기법이란?

**핵심**: "LLM에게 어떻게 질문하느냐"

같은 코드도 질문 방식에 따라 답변이 달라집니다.

## 2. 5가지 기법 비교

| 기법 | F1 | 속도 | 추천 용도 |
|------|----|----|----------|
| Zero-shot | 0.526 | 7초 | 기준선 |
| Few-shot-3 | 0.588 | 7초 | 비용 절감 |
| **Few-shot-5** | **0.615** | **8초** | **일반 사용 ⭐** |
| Chain-of-thought | 0.571 | 24초 | Modern-cpp |
| Hybrid | 0.634 | 33초 | 중요 PR |

---

## 3. Few-shot-5 (추천)

**전략**: 5개 예시 제공

**예시**:
```python
# 예시 1: 메모리 누수
# 예시 2: 버퍼 오버플로우
# 예시 3: 성능 (불필요한 복사)
# 예시 4: 동시성 (데이터 레이스)
# 예시 5: 정상 코드 (이슈 없음)

# 이제 이 코드 분석:
<실제 코드>
```

**장점**:
- 정확도와 속도 균형 ⭐
- 다양한 카테고리 커버
- 프로덕션 검증됨

**단점**:
- Modern-cpp 탐지 어려움 (F1: 0.000)

---

## 4. Hybrid (최고 정확도)

**전략**: Few-shot + Chain-of-thought 혼합

**과정**:
1. Pass 1: Few-shot-5로 광범위 탐지
2. Pass 2: CoT로 Modern-cpp 집중 탐지
3. Pass 3: 중복 제거 + 신뢰도 필터링

**장점**:
- 최고 F1 (0.634)
- Modern-cpp 탐지 가능 (0.250)

**단점**:
- 4배 느림 (33초)
- 2배 비용 (토큰)

**언제 사용?**
- 중요한 PR
- Modern C++ 코드베이스
- 정확도 > 속도

---

## 5. 기법 선택 가이드

```
중요한 PR? 
  YES → Modern C++?
    YES → Hybrid
    NO → Few-shot-5
  NO → 빠른 스캔?
    YES → Few-shot-3
    NO → Few-shot-5
```
